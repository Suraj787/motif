# Motif v3.0.0: Evidence-Grounded Interface Engineering

Motif v3.0.0 introduces an evidence-grounded interface-engineering workflow that can inspect a
running application, retrieve context-appropriate UX evidence, plan and apply a controlled
repair in an isolated worktree, verify the result in a real browser, preserve before-and-after
evidence, and roll back exactly. The bundled Vue workflow is fully browser-verified. Broader
framework, route, authentication, and repair coverage remains experimental.

## Implemented
- UX Evidence Graph: 110 Tier 1-3 claims with sources, limitations, validation, and freshness.
- Deterministic query engine with corrected wildcard semantics: empty applicability dimensions
  are universal, existing values refine ranking, and only `restrict` dimensions are hard
  filters, so universal claims survive rich contexts. `--explain-matching` shows why each claim
  matched or was excluded.
- Claim / finding / enforcement separation: an applicable claim is not a finding. Blocking
  requires detector evidence, sufficient confidence, a machine-supported enforcement mode, and
  no unresolved contradiction. Exposed via `motif evidence evaluate` and the MCP
  `motif.evaluate_findings` tool.
- Browser-verified bundled Vue golden loop (real Chromium in CI, `--require-browser`).
- Isolated git-worktree repair for the colour-only-status class, with before/after evidence,
  exact rollback, and an unchanged baseline.
- Evidence-linked findings; BOSS and external real-project audit support (read-only).
- Aesthetic-convergence (originality) signals: tiered marketing/decorative/structural signals
  with diminishing returns, design-system provenance and product-context weighting, a
  combination gate, and a signal-level breakdown.
- CLI and MCP access to evidence, evaluation, and originality.

## Experimental
- Broad framework repair; arbitrary authenticated applications; complex multi-route crawling.
- Semantic visual comparison; originality across projects; automatic human-usability
  conclusions; repairs beyond the implemented class.

## Motif does not claim
- That all applications can be repaired.
- Full accessibility certification or legal compliance certification.
- AI authorship detection.
- Complete UX correctness, or human usability validation without humans.

## Correctness fixes since beta.1
- Evidence query over-filtering fixed (BOSS 0 -> 105 applicable claims).
- Originality false-saturation fixed (BOSS 100 -> 19; generic-AI fixture still 100).
- Applicable claims separated from violations (BOSS 52 applicability-only "blocking" -> 1
  evidence-backed blocking).

## Validation
- 151 dependency-free self-checks + secret scan; browser golden loop green in CI.
- Audited read-only on BOSS (Vue), a React app, and an AI-generated app; 0 source files
  modified in any.
- See `v3.0.0-release-readiness.md`, `final-v3-real-project-validation.md`,
  `boss-finding-quality-review.md`, and `v3-security-and-safety-review.md`.

`v3.0.0-beta.1` remains unchanged as historical prerelease evidence.
